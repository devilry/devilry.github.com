Devilry

Open source general purpose delivery system

Technical overview

An overview of the technical details of Devilry

Devilry is an open source system for handing in assignment deliveries, and for correcting and providing feedback on the assignments. Devilry uses Django, an open source web application framwork written in Python. Devilry has been planned from the beginning to be a general system for use at educational institutions, supporting all kinds of electronic deliveries.

Devilry has been designed for three groups of users; students, correctors/examiners and administrators. The student interface has been designed to be compatible with most browser, even IE6. The examiner and admin interfaces are designed with more modern web techonolgies requiring modern web browsers to work as expected.

Devilry has a project page at github with a Wiki, issue tracking and git repositories, all of which are publicly available. The Devilry source code is released under the BSD license.

Django

What is Django

Django is a web application framwork written in Python, supported by the Django Software Foundation. It follows the model-template-view pattern, which naturally separates the data (models), web-frontend (template) and the creation of the web-front-end (view). The core Django framwork consists of an object-relation mapper where data models are defined using Python classes, and a database-abstraction API that makes changes to the data models easy.

Templating

The Django template language

The Django template language makes creating HTML code a breeze.

Django's template language is designed to strike a balance between power and ease. It's designed to feel comfortable to those used to working with HTML. Django website

If you have a background in programming, or if you're used to languages like PHP which mix programming code directly into HTML, you'll want to bear in mind that the Django template system is not simply Python embedded into HTML. This is by design: the template system is meant to express presentation, not program logic. Django website

Object-relational mapping

Introduction to the Django ORM

Object-relational mapping (ORM, O/RM, and O/R mapping) in computer software is a programming technique for converting data between incompatible type systems in object-oriented programming languages. Wikipedia

Python classes are used to define data models that the Django ORM converts to relational datadbase models. The Django query langauge hides the SQL queries from the programmer, making a database switch very easy(). The Django query language is database independent, making a database switch very easy. Supported databases are PostgreSQL, MySQL, SQLit, Oracle.

ORM has some overhead compared to native SQL queries, but there are many optimization techniques avaialable in Django. Where maximum efficieny is needed, it's even possible to write RAW SQL statements. (More on optimizing database access in Django.

Database

The Devilry database structure

The database model is a hierarchical tree of nodes. An example for a university:

  • Faculty of Mathematics (node)
    • Department of Informatics (node)
      • INF1000 (subject)
        • Autumn 2010 (period)
          • Oblig1 (assignment)
          • Oblig2 (assignment)
          • Oblig3 (assignment)
          • Oblig4 (assignment)
      • INF1100 (subject)
        • Autumn 2010 (period)
          • Week1 (assignment)
          • Week2 (assignment)
          • Week3 (assignment)
          • Week4 (assignment)
          • Week5 (assignment)
      • INF1010 (subject)
        • Spring 2011 (period)
          • Oblig1 (assignment)
          • Oblig2 (assignment)
          • Oblig3 (assignment)
          • Oblig4 (assignment)

Users can be assigned administrator-rights on nodes, subjects, periods and assignments. Admin rights are inherited. Thus, an administrator on Department of Informatics is also administrator on all subject, periods and assignments below this node.

Data model

Python classes as data models

Data models are defined with Python classes. In this manner, we can define the data model of a relational database in a database independent manner.

Furthermore, we can define arbitrarily complex constraints and custom fields.

Even more powerful is the ability to associate methods with our data models, enabling us to extend the power of relational databases with the flexibility offered by a programming language.

SQL tables

SQL tables created from the Python models

SQL tables are automatically created based on the Python classes.

SQL based relational databases all implement one form of SQL, however their implementations vary. We generate our database from our data model. The database abstraction layer generates the SQL for our database, and handles common sources of compatibility issues, such as index creation and constraints.

Queries using the Django database-abstraction API

The Django database-abstraction API simplifies writing queries greatly, by automatically translating Python code into SQL queries.
